# 09 — UX

> **Purpose:** Specify every UI surface and the moderator experience end-to-end. Verdict Card, Investigation Timeline, Dashboard, Wizard, uncertainty handling. Load when working on Devvit Blocks UI.
>
> **Status:** Living. Structure is stable; copy iterates.

---

## 1. Principles

1. **Investigation transparency over verdict confidence.** Show the work, every time.
2. **Honest uncertainty over confident wrongness.** Low confidence is a feature.
3. **Mod decides; ModPilot recommends.** Nothing auto-fires. Every action button requires an explicit click.
4. **One-glance triage, deep-dive on demand.** 90% of moderator value is in the card; 10% is in the expansion.
5. **Devvit Blocks constraints are creative constraints.** No fighting the platform. Tokens and primitives only.

## 3. Moderator Journey

```
Discover  →  Install  →  Wizard  →  First report  →  Daily queue  →  Override / accept  →  Dashboard review
   (App Directory)  (one-click)  (≤3 min)  (signature moment)   (steady state)         (trust + adapt)        (impact)
```

Three success milestones drive every UX decision:

- **Install → first investigation in under 3 minutes.** Judges time this.
- **First Verdict Card lands within view of expanded Timeline.** The signature moment.
- **First "I'm unsure" verdict appears within first day.** The trust moment.

## 4. Verdict Card (Signature Feature)

The one-glance triage UI. Appears inline on every report in the mod queue.

### 4.1 Layout

```
┌────────────────────────────────────────────────────────────────┐
│  🟥 HIGH RISK    ModPilot recommends: REMOVE         [92% ▲]   │  ← header row
├────────────────────────────────────────────────────────────────┤
│  • 4 reports in 6 min (z=6.2)                       [ev-4]    │  ← top-3 evidence
│  • Author: 3 prior removals (last 30 days)          [ev-2]    │
│  • Thread escalation detected at turn 8             [ev-5]    │
├────────────────────────────────────────────────────────────────┤
│  [ Remove ]  [ Approve ]  [ Escalate ]  [ Lock ]              │  ← action row
│                                       View reasoning ▾         │  ← expansion handle
└────────────────────────────────────────────────────────────────┘
```

### 4.2 Header Row

- **Risk tier pill** (HIGH/MEDIUM/LOW) on the left, color-coded.
- **Recommendation chip** in the center: "REMOVE" / "APPROVE" / "ESCALATE" / "LOCK" / "NO RECOMMENDATION".
- **Confidence badge** on the right showing percentage + tier indicator (▲ HIGH, ● MEDIUM, ▼ LOW).
- LOW-confidence verdicts replace the recommendation chip with `🌱 ModPilot is unsure — your call`.

### 4.3 Evidence Rows

- Exactly 3 rows, drawn from `top_evidence` in the verdict response.
- Each row: bullet, one-line summary, evidence ID chip on the right (`[ev-2]`).
- The IDs are clickable and scroll the timeline to that row.
- If fewer than 3 rows, surface up to N; never pad with placeholder.

### 4.4 Action Row

Four buttons, left-aligned:
- **Primary action** (matching the recommendation) is filled; others are outline.
- HIGH confidence + mod opt-in → primary defaults to focused. Click still required.
- LOW confidence → no button is pre-styled as primary; visual neutrality.
- Cold-start → no button is pre-styled regardless of confidence (per `05-Memory.md`).

Click → confirmation toast → Reddit API call via Devvit → `onModAction` captures alignment.

### 4.5 Expansion

The "View reasoning" handle expands the Investigation Timeline below the card. Animated with Devvit's built-in expand; no custom animation.

### 4.6 States

| State | Visual |
|---|---|
| **Loading** | Skeleton rows with shimmer; "ModPilot is investigating…" |
| **No recommendation (LOW conf)** | 🌱 badge + evidence only; no primary button styling |
| **Cold-start** | Top of card: green "ModPilot is learning your subreddit (X/50)" pill |
| **Partial verdict** | Subtle "Partial investigation — budget reached" footnote |
| **Fallback verdict** | Amber footnote: "Reasoner unavailable; basic signals only" |
| **Validation flagged** | Subtle "ModPilot is unsure about this analysis" badge |
| **Engine unreachable** | Card hidden entirely; native queue continues working |
| **Re-reported** | Top of card: "Re-reported 3 times in 10 min" annotation |
| **Resolved** | Card collapses to a single line: "✓ Removed by u/mod_name 2 min ago" |

### 4.7 Copy Patterns

| Tier | Recommendation phrasing |
|---|---|
| HIGH conf, REMOVE | "ModPilot recommends: **Remove**" |
| HIGH conf, APPROVE | "ModPilot recommends: **Approve**" |
| MEDIUM conf, any | "ModPilot suggests: Remove" |
| LOW conf | "ModPilot is unsure — your call" |
| Cold-start | "ModPilot is learning — surfacing evidence only" |

Recommendation copy is centralized in `ui/copy.ts`. Never inline.

## 5. Investigation Timeline (Signature Expansion)

The expandable reasoning trail. Renders below the Verdict Card.

### 5.1 Layout

```
ModPilot's investigation (took 1.4s)
┌─ ✓ Matched against rules            142ms     [ev-1]
├─ ✓ Checked report velocity            23ms    [ev-4]
├─ ✓ Pulled author history              87ms    [ev-2, ev-3]
└─ ✓ Read thread context              1180ms    [ev-5]

  Verdict (Sonnet • $0.018)
  Author has 3 prior removals [ev-2]. Thread shows escalation
  from turn 8 [ev-5]. Matches Rule 2 [ev-1].

  Confidence: 84% (MEDIUM)
  • LLM self-report: 0.91 (discounted to 0.66)
  • Evidence convergence: 0.88
  • Subreddit accuracy (30d): 0.87
  • Rule-match strength: 0.81
```

### 5.2 Tool Row Shape

- Status icon (✓ success, ✗ failure, ⊘ skipped, ⏱ timeout)
- Tool action verb (past tense): "Matched against rules", "Checked report velocity", "Pulled author history", "Read thread context"
- Latency in ms, right-aligned
- Evidence ID chips, right-aligned

Failures render as: `✗ Read thread context — timeout (Reddit API slow)`. Surface the cause in one line; never raw stack traces.

### 5.3 Verdict Block

- Rationale text with inline `[ev-N]` chips (clickable, scroll the page to that evidence detail).
- Model + cost line beneath (`Sonnet • $0.018`). Toggle via `showCostInDashboard` setting.
- Confidence breakdown shows the four calibration inputs as bullets. This is the "audit trail" judges remember.

### 5.4 Tool Action Verb Map

Internal tool name → user-facing verb. Defined once in `ui/copy.ts`:

| Tool | UI verb |
|---|---|
| `policy_match` | "Matched against rules" |
| `report_velocity` | "Checked report velocity" |
| `user_history` | "Pulled author history" |
| `prior_actions` | "Reviewed prior mod actions" |
| `thread_context` | "Read thread context" |

Never expose raw tool names in the UI.

### 5.5 States

- **Early-stop**: `→ Stopped early — evidence converged at step 3`
- **Budget exit**: `→ Stopped early — budget reached after step 3`
- **Streaming (future)**: rows appear progressively as tools complete. MVP renders atomically.

## 6. Uncertainty UX

The "I'm unsure" moment. The single most demoable trust feature.

### 6.1 When It Triggers

Calibrated confidence < 0.60 (LOW tier). Per `04-InvestigationEngine.md`.

### 6.2 Visual Treatment

- Header recommendation chip replaced with: `🌱 ModPilot is unsure — your call`
- No primary action button styling; all four action buttons visually equal.
- Confidence badge shows the actual percentage with ▼ indicator.
- Evidence still surfaces in full. The moderator gets all the same information; ModPilot just declines to recommend.

### 6.3 Copy

> "I found the following but I'm not confident enough to recommend an action. Your judgment matters here."

(Centralized; never inline.)

### 6.4 Why This Matters

This is what builds trust. A system that says "I don't know" earns the right to be trusted when it says "I do know." Make this surface visibly polished. Practice the demo around this moment.

## 7. First-Run Wizard

Three steps. Under 3 minutes total. Renders as a custom post pinned to the subreddit, mod-only visibility.

### 7.1 Step 1 — Personality (45s)

Single question with three preset cards (Strict / Balanced / Lenient).

Each card shows:
- One-line description
- Two examples ("Reports action at X% confidence")
- Subreddit type that matches ("News, debate, academic" / "General communities" / "Memes, casual")

Default: Balanced. Power-user "Customize" link disabled in MVP (per `05-Memory.md`).

### 7.2 Step 2 — Rules & Region (60s)

- Paragraph input for subreddit rules. Pre-filled from `modwiki` scope if granted.
- Region select: US / EU / UK / IN / Global. Default: Global.
- "Skip for now" link in both fields — wizard advances; can be filled later.

### 7.3 Step 3 — Test Investigation (45s)

- Lists the 3 most recent unactioned reports.
- "Run a test investigation" button.
- On click: spins for ~5s, then renders a real Verdict Card with Investigation Timeline expanded.
- Below: "This is how ModPilot will work in your queue. Done."

This step is the install→first-value moment. It is non-negotiable for the demo.

### 7.4 Resumability

State persisted in `wizard_state:<sub_id>` (Devvit KV, 30d TTL). Mod can close and resume; on next install, "Continue setup" link in the queue surfaces.

## 8. Dashboard

Custom post. Mod-only. The "analytics + queue" home screen.

### 8.1 Top Tiles (One Row, Four Tiles)

```
┌──────────────┬──────────────┬──────────────┬──────────────┐
│ 47           │ 2h 18m       │ 87%          │ $0.43        │
│ investigated │ time saved   │ acceptance   │ today's cost │
│ today        │ today        │ rate         │              │
└──────────────┴──────────────┴──────────────┴──────────────┘
```

Cost tile hidden unless `showCostInDashboard` is enabled.

### 8.2 Tier Breakdown

Horizontal stacked bar showing today's investigations by tier:

```
[████░░░░░░░░░░░░░░░░] 8 FAST  31 STANDARD  8 DEEP
```

### 8.3 Queue Panel

The pending-investigation queue, prioritized by ModPilot:

- One row per pending report.
- Sort: priority score descending (HIGH risk first, then by velocity).
- Each row: target link, mini-verdict (risk tier + recommendation), age, "Open" button → expands Verdict Card.

### 8.4 Footer

- "ModPilot accuracy in your subreddit: 87% (last 30 days)"
- "Avg response time: 3.2s"
- "Cold-start: 23/50 feedback events" (only during cold-start)
- Link to full audit log (post-MVP)

### 8.5 Charts

No external chart libs (Devvit Blocks constraint). Render simple horizontal bars as SVG primitives or styled `vstack`s with width-based fill. Keep it minimal — judges don't need a charting framework, they need legible numbers.

## 9. Menu Action UIs

### 9.1 "Investigate with ModPilot"

Forced investigation on a comment/post. Shows Devvit's native progress UI, then renders a Verdict Card inline.

### 9.2 "Summarize this thread"

Modal with the thread summary structure:
- **Arc** — one sentence
- **Escalation points** — list with turn numbers
- **Instigator candidates** — chip list
- **Off-topic flag** — single line

"From cache" indicator when applicable. ≤500 words.

### 9.3 "Show Moderation Memory"

Modal scoped to current subreddit:
- **Risk tier label** (never raw score)
- **Prior violations** — breakdown by category
- **Recent actions** — last 3 mod actions on the user
- "Wipe this user's memory" mod-only button → confirmation → soft delete

### 9.4 "Explain ModPilot's last call"

Re-renders the existing Verdict Card with Investigation Timeline expanded. Pure Redis read; no Engine call.

## 10. Action Flows

Every action button follows the same pattern.

### 10.1 Remove

```
Mod clicks Remove
   ↓
Inline confirmation chip: "Remove? [Yes] [Cancel]"  (3s timeout → auto-cancel)
   ↓
Reddit API: removeComment / removePost
   ↓
Toast: "Removed. Feedback recorded."  (3s)
   ↓
Verdict Card collapses to resolved state
```

### 10.2 Approve / Escalate / Lock

Identical pattern. The button color and toast copy differ.

### 10.3 Why Inline Confirmation Instead of Modal

Modals break Devvit's queue flow. The inline chip keeps the mod in context and adds friction without blocking. The 3s auto-cancel prevents accidental confirmations.

### 10.4 Failure States

- **Reddit API failure**: toast `"Reddit rejected the action. Try again or do it manually."` Verdict stays in pending state.
- **ModPilot feedback fails to record**: silent retry; no user-facing error. Lost feedback is rare and non-fatal.

## 11. Empty & Error States

### 11.1 Empty States

| Surface | Copy |
|---|---|
| No reports yet | "Nothing in the queue right now. ModPilot is ready when something comes in." |
| No memory for user | "First time we've seen this user in your subreddit." |
| Cold-start with no investigations yet | "ModPilot is set up. We'll surface investigations as reports come in." |
| Wizard not yet started | "Set up ModPilot in under 3 minutes →" |

### 11.2 Error States

| Surface | Copy |
|---|---|
| Engine unreachable (banner) | "ModPilot is temporarily unavailable. Reddit's queue continues working normally." |
| Kill switch active | "ModPilot is paused. Re-enable in settings." |
| Rate-limited (banner) | "ModPilot is throttled for this hour to stay within budget. Investigations resume at HH:MM." |
| Investigation timeout | Card shows partial result + "Investigation timed out — partial evidence below." |
| Reasoner degraded | Card shows fallback verdict + amber footnote "Basic signals only — full reasoning unavailable." |

All error copy is in `ui/copy.ts`. Never inline.

## 12. Cold-Start Badge

Pinned to the top of the Verdict Card, Dashboard, and Settings during cold-start.

```
🌱 ModPilot is learning your subreddit (23 / 50 feedback events).
   Recommendations will become more confident as you provide feedback.
```

Tappable for a brief explanation modal:
> "We use the first 50 of your moderation decisions to calibrate ModPilot to your subreddit's style. During this window, ModPilot is more cautious and never pre-fills action buttons."

Disappears automatically at threshold crossing. Returns if manually reset in settings.

## 13. Accessibility & Copy

- **Color is never the only signal.** Risk tier always shows both color AND icon AND text label.
- **Touch targets ≥44px.** Devvit defaults are compliant; verify on real devices.
- **Copy reading level:** 8th grade. Short sentences. Active voice.
- **Banned UI words (from `Glossary.md`):** RL, policy, reward, training, agent, episode, action space, observation space, trajectory.
- **Preferred UI words:** investigation, evidence, confidence, strategy, recommendation, reasoning, memory.
- **"Memory" never appears as a number.** Only tier labels (new / trusted / neutral / watched).
- **"Confidence" appears with both percentage and tier indicator.** Never percentage alone.

## 15. UX Invariants

1. No action ever fires without an explicit moderator click.
2. Every recommendation shows its evidence; no black-box verdicts.
3. LOW-confidence verdicts never style any action button as primary.
4. Cold-start state is always visible during the cold-start window.
5. The Investigation Timeline is always available (one click) for any verdict.
6. No raw tool names, internal scores, or banned terminology appear in user-facing copy.
7. The mod queue continues working when ModPilot is degraded — never block native moderation flow.
8. All UI copy lives in `ui/copy.ts`. No inline strings.
9. All colors and spacing live in `ui/tokens.ts`. No inline values.

Violating any of these is a UX bug.
